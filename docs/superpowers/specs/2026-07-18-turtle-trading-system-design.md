# 터틀 트레이딩 시스템 설계서 (v1)

작성일: 2026-07-18
상태: 사용자 승인 대기

## 1. 목표

실전 사용 가능한 터틀 트레이딩 시스템. 바이낸스 선물(암호화폐) 대상으로 돈치안 돌파
신호를 자동 감시하고, 텔레그램으로 매수/매도/손절/트레일링 알림을 발송하며,
모바일 웹에서 차트·신호·포지션·뉴스를 확인한다.

근거 문서 (프로젝트 루트 3개):
- [실전 지침서] 터틀 트레이딩 원칙 기반의 체계적 매매 운용 가이드
- 정량적 리스크 관리 전략서: ATR 기반 변동성 통제와 수익 구조의 안정화
- [입문자 가이드] 데이터로 증명된 승리: 터틀 트레이딩 시스템과 프라이스 액션

## 2. 확정된 요구사항 결정

| 항목 | 결정 |
|---|---|
| 대상 시장 | 암호화폐 선물 (바이낸스), 향후 타 시장 확장 고려한 구조 |
| 차트 | TradingView Lightweight Charts (오픈소스). 무료 위젯은 커스텀 오버레이 불가하므로 제외 |
| 타임프레임 | 일봉(1d) + 4시간봉(4h), 심볼·타임프레임별 독립 설정 |
| 감시 종목 | 웹에서 워치리스트 직접 관리 (기본값: BTCUSDT, ETHUSDT) |
| 운영 환경 | 클라우드 (Railway 또는 동급), web + engine 단일 프로젝트 배포 |
| 포지션 관리 | 수동 등록 (진입가·수량 입력) → 등록된 포지션 기준 손절/트레일링/청산 알림 |
| 뉴스/캘린더 | 경제 캘린더 = TradingView 위젯 임베드, 뉴스·속보 = 서버 RSS 수집 + 키워드 필터 |
| 알림 | 텔레그램 봇 (BotFather 생성, 토큰·chat_id 환경변수) |
| 아키텍처 | A안: TypeScript 모노레포 (사용자 확정) |

## 3. 전체 구조

```
turtle-trading/  (TypeScript 모노레포, pnpm workspace)
├─ apps/
│  ├─ web/        Next.js 웹앱 (모바일 우선 반응형)
│  └─ engine/     신호 엔진 워커 (24시간 상주 프로세스)
├─ packages/
│  └─ core/       공유 패키지: 지표 계산, 신호 판정, 포지션 사이징, 타입
├─ data/          SQLite DB 파일
└─ docs/
```

- **packages/core가 단일 진실 공급원(single source of truth).** 웹 차트에 그리는 지표
  값과 텔레그램 알림의 신호 값이 동일한 함수에서 나온다. 값 불일치 원천 차단.
- DB: SQLite (better-sqlite3). 단일 사용자, 파일 백업 용이.
- 배포: Railway 단일 프로젝트에 web / engine 두 서비스, 볼륨에 SQLite 저장.

## 4. 트레이딩 알고리즘

### 4.1 기본 터틀 규칙 (문서 그대로)

파라미터 (심볼·타임프레임별 설정 가능, 아래는 기본값):
- 돈치안 진입 채널: 20봉 / 청산 채널: 10봉
- ATR 기간: 20, 손절 배수: 2.0
- 추세 필터: 200 EMA
- 리스크: 계좌 대비 거래당 2%

판정은 전부 **봉 마감 확정 종가** 기준 (장중 노이즈·휩소 방지).

| 이벤트 | 조건 |
|---|---|
| 롱 진입 | 종가 > 200 EMA AND 종가가 직전 20봉 최고가 돌파 AND 보완 필터 통과 |
| 숏 진입 | 종가 < 200 EMA AND 종가가 직전 20봉 최저가 이탈 AND 보완 필터 통과 |
| 초기 손절 | 롱: 진입가 − 2×ATR(진입 시점) / 숏: 진입가 + 2×ATR |
| 트레일링 | 10봉 반대선이 기존 스톱보다 유리해지면 스톱 상향(롱)/하향(숏) 갱신 |
| 청산 | 롱: 종가가 10봉 최저가 이탈 / 숏: 종가가 10봉 최고가 돌파 |
| 손절 집행 | 보유 포지션의 마크가격이 손절선 도달 — 장중 실시간 감시 (1분 폴링) |

포지션 사이징 (2% 룰): `수량 = (계좌자산 × 리스크%) ÷ (손절폭 = 2×ATR)`
계좌자산은 웹 설정에서 입력·수정.

### 4.2 보완 필터 4종 (사용자 확정: 전부 채택)

전통 터틀 약점(횡보장 휩소, 거래량 무시 가짜 돌파, 200 EMA 후행성, 크립토 특유
과밀 포지션)을 보완. **각 필터는 심볼·타임프레임별 on/off + 임계값 설정 가능.**
진입 신호는 (기본 터틀 조건) AND (활성화된 모든 필터 통과)일 때만 발생.
청산·손절에는 필터를 적용하지 않는다 (나가는 것은 원조 규칙 그대로).

| # | 필터 | 기본값 | 로직 |
|---|---|---|---|
| 1 | ADX 추세강도 | 기간 14, 임계 20 | ADX ≥ 20일 때만 진입 허용. 횡보장 돌파 차단 |
| 2 | 거래량 확인 | 20봉 평균 × 1.5 | 돌파봉 거래량 ≥ 평균×배수일 때만 진입 |
| 3 | Rolling VWAP | 30일 | 롱: 종가 > VWAP, 숏: 종가 < VWAP. 거래량 가중 평균가 = 시장 평균 원가선. 200 EMA 후행성 보완 |
| 4 | 펀딩비 과열 | ±0.1% | 롱: 현재 펀딩비 > +0.1%면 진입 보류. 숏: < −0.1%면 보류. 과밀 청산 위험 회피 |

주: Rolling VWAP은 세션 리셋형 당일 VWAP이 아닌 이동 기간형(기본 30일 = 일봉 30개
/ 4h봉 180개 구간의 거래량 가중 평균). 24시간 크립토 시장에 맞는 형태.

신호 알림에는 각 필터의 통과/차단 상태를 함께 표기한다 (차단된 경우 "돌파 발생했으나
ADX 미달로 진입 보류" 형태의 정보성 알림 — on/off 설정 가능).

### 4.3 신호 상태 머신

심볼·타임프레임별 상태: `FLAT → LONG_SIGNALED → LONG_OPEN → (트레일링 갱신...) → 청산/손절 → FLAT` (숏 대칭).
- LONG_SIGNALED: 진입 신호 발송됨, 사용자 포지션 등록 대기
- LONG_OPEN: 사용자가 포지션 등록 완료 → 이후 손절 실시간 감시 + 트레일링/청산 알림 활성
- 포지션 미등록 상태에서도 청산 신호는 참고용으로 발송

## 5. 신호 엔진 (apps/engine)

- 바이낸스 선물 공개 REST API (`/fapi/v1/klines`, `/fapi/v1/premiumIndex`). 시세 조회만
  하므로 API 키 불필요.
- 스케줄:
  - 봉 마감 감지: 4h/1d 마감 시각(UTC) + 안전 지연 30초 후 확정 캔들 조회 → 신호 판정
  - 손절 감시: 등록 포지션 있을 때만 1분 간격 마크가격 폴링
  - RSS 수집: 10분 간격
- 중복 발송 방지: 신호 유니크 키 = (심볼, 타임프레임, 이벤트 종류, 봉 시각). DB unique 제약.
- 재시작 복구: 마지막 처리 봉 시각을 DB에 기록, 기동 시 누락 봉 소급 판정.
- 자체 경보: 바이낸스 API 연속 실패(5회), 텔레그램 발송 실패 누적 시 "엔진 이상" 텔레그램
  경보 (텔레그램 자체가 죽었으면 DB 로그에 기록, 웹 대시보드에 경고 배지).

## 6. 웹앱 (apps/web)

모바일 우선 반응형. 하단 탭 4개.

### 6.1 차트 탭
- Lightweight Charts: 캔들 + 오버레이
  - 돈치안 진입 채널(20봉 상/하단), 청산 채널(10봉) — 반투명 밴드
  - 200 EMA 라인, Rolling VWAP 라인
  - 과거 신호 마커: ▲ 롱 진입 / ▼ 숏 진입 / ✕ 청산·손절
  - 보유 포지션: 진입가 라인, 현재 손절선(빨간 점선), 트레일링 스톱 이력
- 심볼·타임프레임 전환, 설정 시트에서 해당 심볼 파라미터·필터 수정
- 하단 정보 바: 현재 ATR, ADX, 펀딩비, 돌파까지 거리 %

### 6.2 신호 탭
- 신호 히스토리 (필터 차단 이력 포함)
- 심볼별 현재 상태 카드: 상태 머신 상태, 돌파 근접도, 필터 통과 현황

### 6.3 포지션 탭
- 신호 기반 포지션 등록: 신호 선택 → 실제 진입가·수량 입력
- 2% 계산기: 계좌자산 × 리스크% ÷ (2×ATR) → 권장 수량 표시
- 보유 포지션 목록: 미실현 손익, 현재 스톱, 청산 조건 거리

### 6.4 뉴스 탭
- 경제 캘린더: TradingView Economic Calendar 위젯 임베드
- 뉴스 피드: 서버 수집 RSS (Reuters 마켓, 연합인포맥스, CoinDesk, Cointelegraph 등
  — 소스 목록 설정 파일로 관리)
- 속보 하이라이트: 키워드 매치(트럼프, 관세, 연준, 금리, 전쟁, 지정학 등 — 사용자 편집
  가능) 항목 상단 고정 + 배지. 매치 시 텔레그램 발송 옵션
- 원문 링크로 이동

### 6.5 설정
- 계좌자산, 리스크%, 워치리스트 관리
- 심볼·타임프레임별 파라미터/필터
- 텔레그램 알림 종류별 on/off (진입/청산/손절/트레일링/필터차단/속보/엔진경보)
- 웹 접근 보호: 단일 비밀번호 (환경변수, 세션 쿠키)

## 7. 텔레그램 메시지 형식

```
🟢 롱 진입 신호 — BTCUSDT · 4H
진입가: 65,000
손절가: 63,560 (−2×ATR, ATR20=720)
권장수량: 0.138 BTC (시드 ₩100,000,000 · 리스크 2%)
필터: ADX 28 ✅ | 거래량 2.1× ✅ | VWAP 상방 ✅ | 펀딩 +0.03% ✅
```

트레일링: `🔁 스톱 상향 — BTCUSDT · 4H | 63,560 → 64,900`
청산: `🔴 청산 신호 — 10봉 저점 이탈 종가 마감`
손절: `⛔ 손절선 도달 — 즉시 청산 (마크가격 63,540 ≤ 63,560)`

## 8. DB 스키마 (SQLite, 개요)

- `symbols` (워치리스트), `settings` (전역 설정)
- `params` (심볼×타임프레임 파라미터 + 필터 설정)
- `signals` (신호 이력, 유니크 키 포함, 필터 스냅샷 JSON)
- `positions` (수동 등록 포지션, 상태, 스톱 이력)
- `news_items` (RSS 수집, 키워드 매치 플래그)
- `engine_state` (마지막 처리 봉, 헬스 상태)

## 9. 에러 처리

- 바이낸스 API: 지수 백오프 재시도, rate limit 준수 (공개 API weight 관리)
- 캔들 데이터 무결성: 봉 시각 연속성 검증, 갭 발견 시 재조회
- 텔레그램: 발송 실패 시 재시도 3회, 실패 신호는 DB에 `delivery_failed` 마킹, 웹에 표시
- RSS: 소스별 독립 실패 허용 (한 소스 죽어도 나머지 동작)

## 10. 테스트·검증

- packages/core 유닛 테스트: 돈치안/ATR/EMA/ADX/VWAP 계산을 알려진 데이터 기대값과
  대조. 신호 판정 시나리오 테스트 (돌파, 휩소, 트레일링 갱신 순서 등)
- **백테스트 CLI** (`pnpm backtest`): 바이낸스 과거 데이터로 신호 재생, 필터 조합별
  성과 비교표 출력 (거래 수, 승률, 손익비, MDD). 필터 유효성을 데이터로 확인하는 용도
- 실전 투입 전 1~2주 신호 수신만 하는 관찰 운용 권장

## 11. 범위 제외 (v1에서 안 함)

- 자동 주문 집행 (거래소 API 키 연동 매매) — 알림 전용
- 거래소 잔고 자동 연동
- 다중 사용자 지원
- 한국 주식·미국 주식 (구조만 확장 가능하게: 데이터 소스 인터페이스 분리)

## 12. 구현 순서 (개요)

1. 모노레포 스캐폴드 + packages/core 지표·신호 로직 + 유닛 테스트
2. 백테스트 CLI (로직 검증)
3. engine: 캔들 수집 → 신호 판정 → 텔레그램 발송
4. web: 차트 탭 → 신호/포지션 탭 → 뉴스 탭 → 설정
5. 배포 (Railway) + 실전 관찰 운용

상세 구현 계획은 별도 문서로 작성 (writing-plans 단계).
