## 1. 실행 방법

### 설치

```bash
pnpm install
```

### 로컬 개발

```bash
pnpm --filter @turtle/web dev      # 웹 대시보드, 기본 포트 3000
pnpm --filter @turtle/engine start # 백그라운드 워커 (텔레그램·손절감시·데이터수집)
```

엔진과 웹은 같은 SQLite DB(`DB_PATH`, 기본 `data/turtle.db`)를 공유. 로컬 개발 시 둘 다 켜야 신호/포지션 데이터가 채워짐. 웹만 켜면 UI는 뜨지만 신호가 안 쌓임(엔진이 봉 마감마다 신호를 생성하는 주체).

### 테스트

```bash
pnpm test                        # 전체 유닛 테스트 (core/db/engine)
pnpm --filter @turtle/core test  # 패키지별
```

### 웹 프로덕션 빌드

```bash
cd apps/web && npx next build
```

로컬(Windows)은 symlink 권한 문제로 standalone 출력 비활성. Docker 빌드에서만 `NEXT_STANDALONE=1`로 standalone 활성화(Railway 배포용).

## 2. 웹 대시보드 — 5탭 상세

하단 탭바: 📈차트 · 🔔신호 · 💼포지션 · 📰뉴스 · ⚙️설정 (첫 진입 시 `/chart`로 리다이렉트).

### 📈 차트 (`/chart`)

캔들 차트 + 돈치안 채널/EMA/VWAP 오버레이 + 신호 마커 + 포지션 라인.

**상단 컨트롤**
- 심볼 선택 (워치리스트에서)
- 타임프레임: `1H` / `4H` / `1D` — **1H는 표시 전용**. 코어 전략은 4h/1d만 지원해서 1h 탭엔 자체 파라미터·신호·포지션이 없음. 1h 오버레이는 해당 심볼의 4h 파라미터를 빌려와 그림 — 추세 확인용이지 신호 발생용 아님.
- ⚙ 버튼 — 파라미터 시트 열기 (아래 참고)

**차트 위 통계 바**
| 표시 | 의미 |
|---|---|
| 종가 | 최근 봉 종가 |
| ATR | Average True Range(변동성 지표) 값 |
| ADX | Average Directional Index(추세강도). ≥20이면 녹색, 미만이면 주황 |
| 펀딩 | 현재 펀딩비(%) |
| 돌파까지 | 현재가가 진입 채널 상단까지 남은 거리(%) |

**차트 범례 (선 종류)**
- 진입상단 / 진입하단 — 진입 판정용 돈치안 채널 (녹/적 실선)
- 청산상단 / 청산하단 — 청산(트레일링) 판정용 반대 채널 (녹/적 점선)
- EMA — 추세 필터용 이동평균 (주황)
- VWAP — Rolling VWAP (파랑)

포지션 보유 중이면 진입가(파란 실선)·손절가(빨간 점선) 라인도 표시. 캔들 위 마커: 롱/숏 진입(화살표), 청산(원), 보류(주황 원 = 필터에 막힌 진입).
좌상단 OHLC 호버 리드아웃(O/H/L/C), 우상단 `A` 버튼으로 Y축 자동스케일 리셋.

**⚙ 파라미터 시트** — 심볼×타임프레임별 전략 파라미터 편집. DB(`params` 테이블)에 저장되고 백테스트 `--use-saved-params` 옵션이 이 값을 그대로 씀.

| 그룹 | 필드 | 의미 |
|---|---|---|
| 구조 | 진입 채널(봉) | 돈치안 진입 판정 기간, 기본 20 |
| | 청산 채널(봉) | 반대 채널 트레일링 청산 기간, 기본 15 |
| | ATR 기간 | 변동성 계산 기간, 기본 20 |
| | 손절 배수(×ATR) | 손절폭 = 진입가∓이 배수×ATR, 기본 2.0 |
| | 추세 EMA 기간 | 진입 시 가격이 이 위/아래여야 함, 기본 200 |
| | 리스크%(거래당) | 계좌 대비 거래당 손실 한도, 기본 2% |
| 보완 필터 | ADX 추세강도 | ADX≥임계값일 때만 진입 허용 |
| | 거래량 확인 | 거래량이 평균×배수 이상일 때만 진입 허용 |
| | Rolling VWAP | 가격이 VWAP 방향과 일치할 때만 진입 허용 |
| | 펀딩비 과열 | 펀딩비가 한도(±%) 넘으면 진입 차단 |
| | OI 확인 | 24h 미결제약정 증가 확인(신규 자금 유입=진짜 돌파). **실시간 전용, 백테스트 불가**(무료 API 과거 데이터 30일 한계) |
| 부분 익절 | 익절 시점(R배수) | 이 R배수 도달 시 일부 익절 알림, 기본 1R |
| | 익절 비율(%) | 익절 비중, 기본 50% |
| | 본전 이동 | 부분익절 후 남은 물량 스톱을 진입가로 이동(래칫). 승률↑·MDD↓ 경향, PF 소폭↓ 가능 |
| 타임스톱 | N봉 내 +1R 미도달 시 청산 | 기본 off, 심볼별 opt-in (교차검증상 강한 추세장에서 PF 저하 가능) |

`entryBufferAtr`(진입 버퍼, 기본 0.3×ATR)은 코어에 존재하지만 **이 UI엔 편집 필드 없음** — 기본값 고정 사용.

### 🔔 신호 (`/signals`)

엔진이 감지한 모든 이벤트 히스토리. 상단 엔진 헬스 배지: `엔진 정상`(녹)/`엔진 이상`(적).

**포트폴리오 리스크 카드**
| 필드 | 의미 |
|---|---|
| 오픈 리스크 | 현재 오픈 포지션들의 합산 리스크% / 캡(기본 6%) |
| 방향(롱/숏) | 현재 롱·숏 오픈 포지션 개수 |
| 실현 손익(일/월) | 당일/당월 실현 손익률 |
| 신규 진입 비권장 | 위 지표가 임계 넘으면 표시되는 경고 배지 — **차단 아님, 태그만**. 사유 텍스트 동반 |

**신호 이벤트 종류**
| 코드 | 라벨 | 의미 |
|---|---|---|
| ENTRY_LONG / ENTRY_SHORT | 롱 진입 / 숏 진입 | 돌파 확인, 진입 권장 신호 |
| ENTRY_BLOCKED | 진입 보류 | 돌파는 났으나 필터 미통과 또는 리스크 게이트로 강등 |
| EXIT_LONG / EXIT_SHORT | 롱 청산 / 숏 청산 | 트레일링 채널 반대 터치 |
| TRAIL_UPDATE | 스톱 갱신 | 트레일링 스톱 상향(래칫) 이동 |
| STOP_HIT* | 손절 도달 | 마크가격이 손절가 터치 |
| STOP_NEAR* | 손절 임박 | 손절까지 0.3×initRisk 이내 접근(선경고) |
| PARTIAL_TP* | 부분 익절 | 1R 등 목표가 도달, 일부 익절 권고 |
| VOL_SPIKE | 1분 급변 | 1분봉 변동성 이상 감지(⚡ 경보) |
| TIME_STOP* | 타임스톱 | 옵션 on 시, N봉 내 +1R 미도달 청산 권고 |

각 행에 가격/손절가/스톱 이동 이력/필터별 통과여부(✅/❌)/텔레그램 발송실패 배지(`발송실패`) 표시.

### 💼 포지션 (`/positions`)

**수동 포지션 등록 + 2% 계산기**: 심볼/타임프레임/방향(롱·숏)/진입가/손절가/수량 입력해 수동 등록. 계좌자산(설정 탭에서 입력) 있으면 `2% 룰 권장수량`을 자동 계산해 제시 (`자산×리스크% ÷ (ATR×손절배수)`), `적용` 버튼으로 그대로 채움.

**보유 중 포지션**: 방향 배지·심볼·타임프레임·`청산 처리` 버튼(청산가 입력)·진입가/수량/손절가.

**종료된 포지션**: 최근 20건, 진입→청산가·청산사유.

이 시스템은 **주문을 대신 넣지 않음** — 신호 보고 사용자가 거래소에서 직접 체결 후 여기 수동 등록/청산 기록. 등록해야 엔진의 손절 감시·트레일링·부분익절 알림이 그 포지션을 추적함.

## 3. 백테스트 방법

```bash
# 1) 필터 조합 비교표 — 구조 파라미터(진입/청산/손절/버퍼)는 클래식 고정, 필터 on/off 조합만 비교
pnpm backtest BTCUSDT 1d 2022-01-01

# 2) 파라미터 그리드 스윕 — 진입/청산/손절/버퍼/부분익절 조합 전수 비교, PF/승률 상위 12개 출력
pnpm backtest:sweep BTCUSDT 4h 2023-01-01 [종료일]

# 3) 교차검증 — 채택 후보 파라미터 vs 기본값(baseline)을 다심볼·다기간에서 비교
pnpm backtest:crossval

# --use-saved-params: 웹 차트 ⚙ 파라미터 시트에서 저장한 심볼별 DB 값 사용
pnpm backtest BTCUSDT 4h 2024-01-01 --use-saved-params       # 단일 결과 출력(비교표 아님)
pnpm backtest:sweep BTCUSDT 4h 2023-01-01 --use-saved-params # 저장된 필터 위에서 구조만 스윕
pnpm backtest:crossval --use-saved-params                    # 저장_PF/승률/MDD 컬럼 추가
```

`--use-saved-params`도 funding/OI 필터는 강제 off(과거 데이터 부재 제약은 백테스트 전체에 항상 적용).

**결과 표 읽는 법** — 헤더 의미와 중요도는 이전 세션 논의 참고(간단 재정리):
- **PF(Profit Factor)** = 총이익÷총손실. **가장 중요한 지표**, >1이면 기대값 양수.
- **MDD(Max Drawdown, %)** = 최대 낙폭. 낮을수록 좋음, PF 다음으로 중요.
- **평균R** = 거래당 평균 손익을 R(리스크 1단위=진입가~손절가 거리) 배수로 표현.
- **승률(%)** = 이익 거래 비율. 터틀은 저승률·고손익비 설계라 **목표로 삼지 말 것**.
- **거래수** = 표본 크기. 10건 미만은 스크립트가 자동 제외(통계 신뢰도 부족).
- **최종** = 백테스트 종료 시점 계좌 잔고(복리 반영, PF와 반드시 비례하진 않음).

**주의**: 스윕 상위 결과는 단일심볼·단일기간 — 과최적화 위험. 반드시 `backtest:crossval`로 여러 심볼/기간에서 재검증 후 채택. 비용(`DEFAULT_COSTS`: taker 0.05%+슬리피지 0.05%, 편도) 반영된 결과라 실전에 더 가까움 — 단, `runBacktest` 순수함수 자체 기본값(유닛테스트용)은 무비용.

## 4. 용어집

| 용어 | 풀네임 | 의미 |
|---|---|---|
| 돈치안 채널 | Donchian Channel | 직전 N봉 고가/저가로 만든 채널. 돌파 시 진입/청산 판단 |
| ATR | Average True Range | 평균 실질 변동폭. 손절폭·포지션 사이징 기준 |
| ADX | Average Directional Index | 추세 강도 지표. 값이 높을수록 추세 뚜렷 |
| EMA | Exponential Moving Average | 지수이동평균. 여기선 200봉 장기추세 필터 |
| VWAP | Volume Weighted Average Price | 거래량가중평균가. Rolling 방식으로 최근 N일 기준 방향 필터 |
| OI | Open Interest | 미결제약정. 선물 시장 전체 포지션 총량 — 증가 시 신규 자금 유입 신호 |
| R | Risk unit | 진입가~손절가 거리를 1단위로 정규화한 손익 표현. +2R = 리스크의 2배 이익 |
| PF | Profit Factor | 총이익÷총손실. 백테스트 핵심 평가지표 |
| MDD | Maximum Drawdown | 자산곡선 고점 대비 최대 낙폭(%) |
| 부분 익절 | Partial Take-Profit | 목표 R 도달 시 포지션 일부만 먼저 청산 |
| 본전 이동 | Move Stop to Breakeven | 부분익절 후 잔여 물량 스톱을 진입가로 이동 |
| 타임스톱 | Time Stop | 일정 봉수 내 목표 미도달 시 강제 청산 |
| 트레일링 스톱 | Trailing Stop | 유리한 방향으로만 이동하는 손절(래칫) |
| 진입 강등 | Entry Demotion | 필터/리스크게이트 미통과 시 신호는 발송하되 "비권장" 태그만 붙임(차단 아님) |
| 크로스밸리데이션 | Cross-validation | 후보 파라미터를 여러 심볼·기간에 적용해 과최적화 여부 검증 |

## 5. 환경변수 / 배포

| 변수 | 설명 |
|---|---|
| `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` | 텔레그램 알림용. 미설정 시 알림 생략(신호는 DB엔 기록됨) |
| `WEB_PASSWORD` | 웹 로그인 비밀번호. 미설정 시 인증 없음(로컬 전용) |

**배포**: Railway, Dockerfile 자동빌드. Volume `/data` 마운트 + 위 환경변수 4개 설정. `railway.json`에 `restart: ALWAYS` + healthcheck `/login`. 단일 컨테이너에서 web+engine이 SQLite 볼륨 공유.

텔레그램 봇 생성은 BotFather로 사용자가 직접(토큰 발급 후 env에 넣기).

## 6. 운영 체크리스트 / 트러블슈팅

- **신호가 하나도 안 쌓임** → 엔진(`@turtle/engine`)이 안 돌고 있는지 확인. 웹만 켜면 신호 생성 안 됨.
- **차트 값과 알림 값이 다름** → 있으면 안 되는 버그. 지표 계산은 `packages/core`가 유일한 진실 공급원 — 웹/엔진 어디도 재구현 안 함이 원칙이라 이 버그 나면 core 함수 참조 안 하고 어디선가 재구현했는지 의심.
- **OI/funding 필터를 백테스트에서 켰는데 결과가 이상함** → 정상. 무료 API 과거 데이터 30일 한계로 백테스트는 이 필터들 강제 off. 실시간 엔진에서만 opt-in 동작.
- **부분 익절인데 실제 거래소에서 안 팔림** → 정상 동작. 부분익절은 **알림 전용**, 실제 분할 청산은 사용자가 거래소에서 직접. 본전 이동만 예외로 엔진이 DB 스톱값을 자동 갱신.
- **1H 탭에서 신호/포지션 안 보임** → 정상. 1H는 표시 전용, 코어 전략은 4h/1d만 지원.
- **실전 투입 전** → 1~2주 신호만 관찰 운용 권장(CLAUDE.md 명시). 스윕 상위 결과 바로 채택 금지 — crossval 필수.
