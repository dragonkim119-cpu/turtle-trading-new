# 보안 리뷰

최종 점검: 2026-07-19. 대상: 웹앱(인증·API·입력검증), 엔진(시크릿·외부호출), 저장소(시크릿 노출).

## 요약

이 시스템은 **거래 실행 능력이 없다**(바이낸스 공개 시세 API만 사용, 거래 API 키 없음).
따라서 최악의 침해도 "신호·설정 열람" 수준이며 **자금 탈취는 구조적으로 불가능**하다.
실전 배포 시 `WEB_PASSWORD`를 강하게 설정하는 것 하나가 가장 중요하다.

## 점검 완료 — 안전

| 항목 | 상태 |
|---|---|
| API 인증 | 8개 라우트 전부 `requireAuth`로 서버측 HMAC 세션 쿠키 검증 |
| 시크릿 관리 | 텔레그램 토큰·비밀번호 모두 환경변수. 코드 하드코딩 없음 |
| git 노출 | `.env`, `data/*.db` gitignore 처리, 추적 안 됨(확인) |
| 자금 위험 | 거래 API 키 없음. 공개 시세만. 알림 전용 |
| XSS | React 기본 이스케이프. `dangerouslySetInnerHTML` 미사용. 뉴스 링크는 외부 원문으로만 이동 |
| 쿠키 | httpOnly + sameSite=lax + secure(프로덕션). CSRF 상당 방어 |
| 입력 검증 | 워치리스트·포지션·파라미터·candles 라우트 검증 있음 |

## 수정 완료 (커밋 efbae42)

| 취약점 | 위험 | 조치 |
|---|---|---|
| 로그인 비밀번호 `!==` 비교 | 타이밍 사이드채널 | sha256 해시 후 `timingSafeEqual` 상수시간 비교 |
| candles `symbol`/`tf` 미검증 | 업스트림 URL 파라미터 인젝션 | 정규식 검증(`[A-Z0-9]{5,20}`, tf 화이트리스트) + limit 클램프 |

## 감수된 위험 (단일 사용자 도구 전제 — 배포 시 인지)

| 항목 | 설명 | 완화 |
|---|---|---|
| `WEB_PASSWORD` 미설정 시 완전 개방 | 로컬 편의용 — 미설정이면 인증 없음 | **배포 시 반드시 설정**. 안 하면 URL 아는 누구나 접근 |
| 로그인 레이트 리밋 없음 | 무차별 대입 방어 없음 | 강한 비밀번호(16자+)면 실질 위험 낮음. 필요시 Cloudflare 등 앞단 배치 |
| 비밀번호가 세션키 파생 원천 | 약한 비밀번호면 세션토큰 유추 가능(단 httpOnly 쿠키) | 강한 비밀번호로 해소 |

## 배포 체크리스트

- [ ] `WEB_PASSWORD`를 16자 이상 무작위로 설정
- [ ] HTTPS 도메인으로만 접속(Railway 기본 제공) — secure 쿠키 활성
- [ ] 텔레그램 토큰·chat_id는 환경변수로만 주입, 로그·커밋에 남기지 말 것
- [ ] 거래소 거래 API 키는 **이 시스템에 절대 넣지 말 것** (필요 없음, 넣으면 공격면만 증가)
- [ ] `data/` 볼륨(SQLite)은 공개 경로에 노출하지 말 것
