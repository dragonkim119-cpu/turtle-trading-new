# 활용 가이드 — 터틀 트레이딩 시스템

이 문서는 시스템을 **실전에서 어떻게 쓰는가**를 다룬다. 설치·배포는 [README](../README.md), 설계는 [docs/superpowers/specs](superpowers/specs) 참고.

---

## 1. 이 시스템이 하는 일 / 안 하는 일

**한다**
- 바이낸스 선물 시세를 봉 마감마다 감시 → 터틀 규칙(돈치안 돌파 + 200EMA + 보완필터)으로 진입/청산 신호 판정
- 신호 발생 시 텔레그램 알림 (진입가·손절가·2% 룰 권장수량 포함)
- 보유 포지션의 손절선 도달·트레일링 갱신·부분익절 1R 도달을 실시간 감시·알림
- 모바일 웹에서 차트(신호 마커·채널·손절선)·신호이력·포지션·거시뉴스·경제캘린더 확인
- 거시 뉴스/속보 키워드(트럼프·관세·연준 등) 수집·하이라이트·텔레그램 발송

**안 한다 (의도적)**
- 자동 주문 실행 ❌ — 알림만. 실제 매수/매도/분할청산은 **본인이 거래소에서 직접**
- 거래소 잔고 자동 연동 ❌ — 포지션은 수동 등록
- 투자 자문 ❌ — 시스템은 규칙 기반 신호 도구일 뿐, 매매 판단·책임은 본인

## 2. 핵심 개념 3분 요약

- **진입**: 종가가 20봉 최고가를 (0.3×ATR 버퍼 넘어) 돌파 + 200EMA 위 + 켜둔 필터 전부 통과 → 롱. (숏은 대칭)
- **손절**: 진입가 − 2×ATR. 이게 곧 1R(리스크 1단위). 장중 마크가격이 여기 닿으면 즉시 청산 알림
- **수량(2% 룰)**: 계좌자산 × 2% ÷ (2×ATR). 한 번 틀려도 자산의 2%만 잃음
- **청산**: 종가가 15봉 반대채널 이탈. 그전까진 트레일링 스톱이 유리한 방향으로만 따라 올라감
- **부분익절**: 가격이 1R 도달 시 절반 익절 권고 → 승률 개선. (옵션: 남은 물량 스톱 본전 이동)
- **철학**: 저승률(30~50%)·고손익비. 승률 낮아도 큰 추세 한 번이 여러 손실 상쇄 → **기대값(PF>1)이 목표**. 연속 손절은 정상이니 시스템 신뢰가 관건

## 3. 활용 시나리오

### 시나리오 A — "신호만 받아보기" (실전 투입 전 1~2주, 강력 권장)
1. 엔진·웹 배포, 텔레그램 봇 연결, 워치리스트에 관심 심볼 등록
2. 포지션은 등록하지 않고 **진입 신호 알림만 관찰**
3. 실제로 그 신호대로 들어갔다면 어땠을지 눈으로 추적 → 시스템·심리 적응
4. 백테스트 수치(승률·PF)와 실제 신호 빈도가 감각적으로 맞는지 확인

### 시나리오 B — 실전 운용 (반자동)
1. **진입 신호** 텔레그램 수신 → 판단 후 거래소에서 실제 진입
2. 웹 **포지션 탭**에서 실제 진입가·수량 등록 (2% 계산기로 권장수량 확인)
3. 이후 시스템이 자동 감시:
   - 🟡 **1R 도달** → 절반 익절 권고 알림 → 거래소에서 절반 청산
   - 🔁 **트레일링 스톱 갱신** → 손절 라인 올려라 알림
   - ⛔ **손절선 도달** → 즉시 청산 알림
   - 🔻 **청산 신호**(15봉 이탈) → 전량 종료 알림
4. 실제 청산하면 웹에서 **청산 처리** → 포지션 종료 기록

### 시나리오 C — 파라미터 튜닝·검증 (매매 안 하는 시간에)
1. `pnpm backtest SYMBOL TF START` — 필터 조합별 성과 비교
2. `pnpm backtest:sweep SYMBOL TF START` — 진입/청산/손절/버퍼/부분익절 그리드 스윕 → 상위 조합 발굴
3. `pnpm backtest:crossval` — 후보 파라미터를 여러 심볼·기간에 교차검증(과최적화 방지)
4. 여러 데이터셋에서 안정적으로 좋은 조합만 → 웹에서 해당 심볼 파라미터로 반영

### 시나리오 D — 거시 이벤트 대응
- **뉴스 탭**: 트럼프·관세·연준·금리 등 키워드 속보 상단 고정 + 경제 캘린더(발표 일정)
- 중요 지표 발표(FOMC 등) 전후엔 변동성 급증 → 신규 진입 보류하거나 손절 여유 조정 판단 근거로

## 4. 화면별 사용법

| 탭 | 쓰는 법 |
|---|---|
| **차트** | 심볼·타임프레임(4H/1D) 전환. ⚙로 해당 심볼 파라미터·필터·부분익절 편집. 하단바에 ATR/ADX/펀딩/돌파거리% |
| **신호** | 최근 신호 이력. 진입 보류(필터차단) 이유도 표시. 엔진 상태 배지(정상/이상) |
| **포지션** | 진입가·수량 등록, 2% 권장수량 자동계산·적용. 보유 포지션 손절 현황. 청산 처리 |
| **뉴스** | 키워드 속보 + 거시뉴스 피드 + 경제 캘린더(TradingView) 임베드 |
| **설정** | 계좌자산, 워치리스트 추가/제거, 알림 종류별 on/off, 속보 키워드 편집 |

### 파라미터 편집 팁 (차트 ⚙)
- **필터는 심볼마다 다르게** — 백테스트 결과가 심볼·장세별로 갈림. 잡코인은 거래량 필터·본전이동이 유리한 경향, BTC는 필터 적을수록 나은 경우도
- 바꾸기 전 **백테스트로 확인** 습관화. 감으로 만지지 말 것(원조 터틀 정신)

## 5. 운용 수칙 (문서 3종의 핵심)

1. **기계적 복종** — 신호 뜨면 예측 말고 규칙대로. 자만이 파멸
2. **손절은 운영비** — 실패가 아니라 사업 비용. 감정 배제하고 집행
3. **연패는 정상** — 9번 지고 1번 크게 먹는 구조. 승률에 좌절 말 것
4. **2% 룰 절대 준수** — 수량 키우고 싶은 유혹이 계좌 파괴 1순위
5. **실전 전 관찰 운용** — 시나리오 A를 건너뛰지 말 것

---

## 6. 주의·한계

- **부분익절·본전이동은 알림 전용** — 시스템이 실제 분할청산은 안 함(본전 스톱값 DB 갱신은 함). 거래소에서 직접 실행
- **신호는 봉 마감 후** — 4시간봉이면 최대 4시간 대기. 휩소 방지 위한 의도된 지연(장중 손절 감시만 실시간)
- **스윕 상위 조합 과최적화 주의** — 반드시 교차검증 후 채택
- **투자 책임은 본인** — 이 시스템은 도구. 손익 책임은 사용자에게 있음
